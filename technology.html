A stack chosen for institutions, not startups

Every technology in HARDAM was picked for one reason: it still works when a network is old, an IT staff is small, and a system needs to run for a decade.

JV

Java 25

Powers the counter and kiosk desktop clients — stable, long-supported, and easy to deploy on aging institutional hardware.

SW

Java Swing

The desktop UI toolkit behind the counter application — lightweight and dependency-free for IT teams to install.

PHP

PHP

Drives the API layer — widely supported by the shared hosting most institutions already use.

SQL

MySQL

Stores tickets, staff accounts, and historical reporting data with a proven backup and replication story.

H5

HTML5

Structures the public portal, kiosk screens, and admin dashboard.

C3

CSS3

Delivers the responsive, accessible interface across desktop, kiosk, and mobile.

JS

JavaScript

Handles live updates, counters, and interactivity in every browser-based screen.

WS

WebSocket

Keeps ticket boards and dashboards updated in real time without polling the server.

API

REST API

A documented, versioned interface for integrating HARDAM with existing student, hospital, or billing systems.
